# cv_auto_detection
# Прототип системы детекции уборки столиков по видео

Проект демонстрирует работу простого пайплайна компьютерного зрения для автоматического отслеживания состояния столика (пусто / занято) на заданном видео. Система выделяет область одного столика (ROI), с помощью YOLOv8 определяет наличие человека в этой области и фиксирует события «подход» и «уход». На основе временных меток вычисляется среднее время между уходом гостя и следующим подходом (потенциальное время уборки).

## Возможности

- Ручной выбор области столика на первом кадре.
- Детекция людей с помощью предобученной модели YOLOv8 (класс `person`).
- Отслеживание состояний:
  - `empty` – в области столика нет людей.
  - `occupied` – в области столика есть человек.
- Фиксация событий:
  - **подход** – переход из `empty` в `occupied`.
  - **уход** – переход из `occupied` в `empty`.
- Расчёт временных интервалов между уходом и следующим подходом.
- Сохранение видео с визуализацией (прямоугольник ROI меняет цвет в зависимости от состояния).
- Генерация отчётов: `events.csv` (все события) и `report.txt` (статистика).

## Требования

- Python 3.8 – 3.11 (рекомендуется 3.10 или 3.11, так как PyTorch может не поддерживать Python 3.12+).
- Установленные библиотеки из `requirements.txt`.

## Установка

1. Клонируйте репозиторий или скопируйте скрипт `main.py` и файл `requirements.txt`.

2. Создайте виртуальное окружение (рекомендуется) и установите зависимости:

```bash
pip install -r requirements.txt
```

## Использование

Запустите скрипт, указав путь к видеофайлу:
python main.py --video <путь к видеофайлу>


## Процесс работы
1) После запуска откроется окно с первым кадром видео.

2) Нажмите и перетащите мышкой прямоугольник вокруг столика, который вы хотите отслеживать.

3) Нажмите Enter или пробел для фиксации выделения.

4) Начнётся обработка видео. В консоли будут выводиться временные метки событий.

5) После завершения обработки появятся файлы:
   
- output.mp4 – видео с отрисованным ROI (зелёный – пусто, красный – занято).

- events.csv – таблица событий (время, тип события, состояния).

- report.txt – текстовый отчёт со средней задержкой и списком задержек.

## Логика работы

- Выбор ROI – пользователь вручную задаёт прямоугольник, в котором будет анализироваться присутствие людей.

- Детекция человека – на каждом кадре YOLOv8 обнаруживает всех людей (class=0). Если какой-либо bounding box пересекается с ROI, считается, что столик занят.

- Изменение состояния – система запоминает текущее состояние (empty/occupied). При изменении фиксируется событие с временной меткой (в секундах от начала видео).

- Расчёт задержек – после обработки всех кадров для каждого ухода ищется следующий подход. Разница во времени считается задержкой.

- Визуализация – на каждом кадре рисуется прямоугольник ROI, цвет которого меняется в зависимости от состояния. Дополнительно отображается текст Empty или Occupied.

## Ограничения
- Обработка ведётся на всём кадре, что может быть медленно для высокого разрешения. Для ускорения можно обрезать кадр до ROI.

- Модель YOLOv8n не различает гостей и сотрудников – любое появление человека в ROI считается подходом.

- Требуется хорошее освещение и видимость людей.
